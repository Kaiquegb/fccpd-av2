# Projeto 2 – Docker e Microsserviços

Este repositório contém a solução dos 5 desafios do Projeto 2, organizados em pastas separadas (`desafio1` a `desafio5`), cada uma com seu próprio código, Dockerfiles, arquivos de orquestração (quando necessário) e README específico.

## Estrutura do Repositório

- `/desafio1` – Containers em rede, comunicação via HTTP em uma rede Docker customizada.
- `/desafio2` – Volumes e persistência de dados com banco em container.
- `/desafio3` – Orquestração de múltiplos serviços usando Docker Compose.
- `/desafio4` – Dois microsserviços independentes se comunicando via HTTP.
- `/desafio5` – Arquitetura de microsserviços com API Gateway centralizando o acesso.

## Como Executar

Cada desafio possui seu próprio README com:

- Descrição da solução, arquitetura e decisões técnicas.  
- Explicação detalhada do funcionamento (containers, rede, microsserviços, fluxos).
- Instruções de execução passo a passo (como subir os containers e testar).

Acesse a pasta de cada desafio e siga as instruções específicas.

## Avaliação

A solução foi desenvolvida levando em conta os critérios descritos na orientação do projeto:

- Funcionamento técnico (40%).
- Explicação no README (30%). 
- Organização e boas práticas (20%).
- Originalidade / autoria (10%).
